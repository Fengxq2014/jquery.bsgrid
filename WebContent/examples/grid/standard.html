Documention:

1, th properties:
   w_index: column index
   w_render: use js code, it will use eval(code) to execute it. To use data index, it must like ${IndexName}, and note that here this indexName can not contains '}'
     Note: if column has property w_render, then ignore property w_index. It's be allowed to no property w_index or w_render
   w_align: body column align, not header th
   w_sort:  like "id", "id,desc", "id,asc"
   w_length: if column's value length longer than it, auto sub and title it
             sub: txt.substring(0, MaxLength-3) + '...'
             if column's render is not false, then this property is not make effective to it
   w_title: if set "true", then will tip column, ignore if it's length is longer than w_length

2, $.fn.bsgrid.defaults.settings or $.fn.bsgrid.init(gridId, settings) second param settings
   url, page request url
   dataType, 'json', 'xml', default 'json'
   autoLoad, load onReady, default true
   pageAll, display all datas, no paging only count, default false
   pageSize, if set value little then 1, then pageAll will auto set true, default 20
   pageSizeSelect, if display pageSize select option, use with property pageSizeForGrid, default false
   pageSizeForGrid: pageSize select option
   longLengthAotoSubAndTitle: if column's value length longer than it, auto sub and title it, default true
                                 sub: txt.substring(0, MaxLength-3) + '...'
                                 if column's render is not false, then this property is not make effective to it

3, process lock screen, global setting
   $.fn.bsgrid.defaults.lockScreen, user lock screen method
   $.fn.bsgrid.defaults.unlockScreen, user lock screen method
   If modify these settings bellow, please see source first, then you should not modify it to avoid incorrect.
     $.fn.bsgrid.defaults.isProcessLockScreen, default true
     $.fn.bsgrid.defaults.beforeSend, process page beforeSend method, default lock screen
     $.fn.bsgrid.defaults.complete, process page complete method, default unlock screen

4, th properties default:
   $.fn.bsgrid.defaults.settings.colsProperties, properties follow:
     body row column config: align, maxLength
     config properties's name: indexAttr, sortAttr, alignAttr, lengthAttr, renderAttr, titleAttr

5, $.fn.bsgrid.defaults.pagingToolbarAlign, global setting
     paging toolbar align: left, center, right, default right
   $.fn.bsgrid.defaults.pagingBtnClass, global setting
     paging toolbar button css class, default 'pagingBtn' in grid.css
   How to Change Paging Toolbar: global setting
     $.fn.bsgrid.addPagingToolbar(options)
     $.fn.bsgrid.setPagingToolbarEvents(options)
     $.fn.bsgrid.dynamicChangePagingButtonStype(options)

6, displayPagingToolbarOnlyMultiPages
   $.fn.bsgrid.defaults.displayPagingToolbarOnlyMultiPages, true or false, default false means alway diaplay pagingToolbar

7, displayBlankRows
   $.fn.bsgrid.defaults.displayBlankRows, true or false, default true
   
8, stripe rows
   $.fn.bsgrid.defaults.stripeRows, true or false, default false, css: grid.css -> '.bsgrid tr.even_index_row'

9, request post params names, global setting
   to modify that: $.fn.bsgrid.defaults.requestParamsName.X = '', X may pageSize, curPage, sortName, sortOrder

10, skins
   in addition to grid.css, other need skins/grid_*.css
